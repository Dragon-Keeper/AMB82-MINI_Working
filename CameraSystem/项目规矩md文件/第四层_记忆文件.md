# 第四层：记忆文件

> **层级定位**：临时记忆层，记录会话上下文和个性化信息
> **加载范围**：单次会话或特定项目
> **核心作用**：存储硬件信息、引脚配置、库使用记录、问题解决方案

---

## 一、定义与定位

记忆文件是AI Agent规则体系的最底层，用于存储临时记忆、硬件配置、库使用记录和问题解决方案。与上层规则文档不同，记忆文件的内容是动态变化的，随着会话和项目进展不断更新。记忆文件弥补了AI Agent短期记忆的局限性，为长期协作提供上下文支持。

### 1.1 核心特征

| 特征 | 说明 |
|------|------|
| **动态性** | 内容随会话和项目进展动态变化 |
| **临时性** | 主要服务于当前会话或项目阶段 |
| **个性化** | 记录用户偏好、硬件配置等个性化信息 |
| **补充性** | 补充上层规则文档未覆盖的细节 |

### 1.2 为什么需要记忆文件

AI Agent存在短期记忆限制，主要体现在以下几个方面：

| 限制 | 说明 | 记忆文件的解决方案 |
|------|------|-------------------|
| **上下文窗口有限** | 无法记住所有历史对话 | 记录关键信息供后续参考 |
| **会话隔离** | 新会话无法访问旧会话信息 | 持久化存储跨会话信息 |
| **硬件配置丢失** | 每次需要重新说明硬件连接 | 记录引脚分配和接线信息 |
| **库版本混乱** | 忘记使用的库和版本 | 记录库依赖和版本信息 |
| **问题重复出现** | 相同问题反复遇到 | 记录问题解决方案 |

### 1.3 与上层规则的关系

| 关系 | 说明 |
|------|------|
| **遵循** | 遵循上层规则文档的约束 |
| **补充** | 补充上层规则未覆盖的细节 |
| **临时** | 提供临时性的上下文支持 |
| **可选** | 根据需要选择使用 |

- 85°C是上电默认值
- 可能是转换未完成就读取
**解决方案**:

- 添加数据有效性检查
**代码修改**:
```cpp
// 修改前

// 修改后
sensors.requestTemperatures();
delay(100);  // 等待转换完成
float temp = sensors.getTempCByIndex(0);
if (temp == 85.0) {
  temp = sensors.getTempCByIndex(0);  // 重新读取
}
```

**问题描述**: 屏幕刷新时出现闪烁
**原因分析**: 
- 每次都清屏再重绘
- 刷新频率过高
**解决方案**: 
- 只更新变化的区域
- 降低刷新频率到10Hz
**版本**: v1.01

#include <Adafruit_SSD1306.h>

#define SCREEN_WIDTH 128
#define SCREEN_HEIGHT 64
Adafruit_SSD1306 display(SCREEN_WIDTH, SCREEN_HEIGHT, &Wire, -1);

  display.begin(SSD1306_SWITCHCAPVCC, 0x3C);
  display.clearDisplay();

### DS18B20
```cpp
#include <OneWire.h>

void setup() {
  sensors.begin();
}

void loop() {
  sensors.requestTemperatures();
  float temp = sensors.getTempCByIndex(0);
}

### RTClib (DS3231)
```cpp
#include <RTClib.h>

RTC_DS3231 rtc;

void setup() {
  rtc.begin();
  if (rtc.lostPower()) {
    rtc.adjust(DateTime(F(__DATE__), F(__TIME__)));
  }

  DateTime now = rtc.now();
  Serial.print(now.year());
  Serial.print("/");
  Serial.print(now.month());
  Serial.print("/");
  Serial.println(now.day());
}

### 2.4 问题解决记录 (Solutions.md)

问题解决记录文件记录遇到的问题和解决方案，便于后续参考。

#### 2.4.1 文件结构

```markdown
# Solutions.md - 问题解决记录

## 编译问题

### 问题: 找不到头文件
**错误信息**: `fatal error: Adafruit_SSD1306.h: No such file or directory`
**原因**: 库未安装
**解决**: 通过库管理器安装 Adafruit_SSD1306

### 问题: 内存不足
**错误信息**: `section `.text' will not fit in region `iram0_0_seg'`
**原因**: 代码过大或库占用过多
**解决**: 
1. 移除未使用的库
2. 使用F()宏存储字符串
3. 优化代码结构

## 运行时问题

### 问题: ESP32反复重启
**现象**: 设备启动后不断重启
**原因**: 
1. 电源不足
2. 代码异常
3. 看门狗超时
**解决**: 
1. 检查电源供电能力
2. 添加串口调试输出
3. 增加看门狗喂狗

### 问题: I2C设备无响应
**现象**: OLED或RTC初始化失败
**原因**: 
1. 接线错误
2. I2C地址错误
3. 上拉电阻缺失
**解决**: 
1. 检查SDA/SCL接线
2. 使用I2C扫描程序确认地址
3. 添加4.7K上拉电阻

### 问题: DS18B20返回85°C
**现象**: 温度传感器返回固定值85°C
**原因**: 
- 85°C是上电默认值
- 转换未完成就读取
**解决**: 
```cpp
sensors.requestTemperatures();
delay(100);  // 等待转换完成
float temp = sensors.getTempCByIndex(0);
```

### 问题: 编码器计数不准
**现象**: 旋转编码器计数时多时少
**原因**: 机械抖动
**解决**: 
```cpp
// 软件消抖
int read_encoder() {
  static unsigned long last_time = 0;
  unsigned long now = millis();
  
  if (now - last_time < 10) {  // 10ms消抖
    return 0;
  }
  last_time = now;
  
  // 读取编码器...
}
```

## 硬件问题

### 问题: GPIO无法控制
**现象**: 引脚输出无效
**原因**: 
1. 引脚模式未设置
2. 使用了启动关键引脚
**解决**: 
1. 确保调用 `pinMode()` 设置模式
2. 避免使用GPIO0, GPIO2, GPIO12, GPIO15

### 问题: ADC读数不准
**现象**: 模拟输入值波动大
**原因**: 
1. ADC精度限制
2. 电源噪声
**解决**: 
1. 多次采样取平均
2. 添加滤波电容
```cpp
int read_adc_avg(int pin, int samples) {
  long sum = 0;
  for (int i = 0; i < samples; i++) {
    sum += analogRead(pin);
    delay(1);
  }
  return sum / samples;
}
```

## WiFi问题

### 问题: WiFi连接失败
**现象**: 无法连接到WiFi
**原因**: 
1. SSID/密码错误
2. WiFi信号弱
3. 路由器限制
**解决**: 
```cpp
WiFi.begin(ssid, password);
int retry = 0;
while (WiFi.status() != WL_CONNECTED && retry < 20) {
  delay(500);
  Serial.print(".");
  retry++;
}
if (WiFi.status() == WL_CONNECTED) {
  Serial.println("WiFi连接成功");
} else {
  Serial.println("WiFi连接失败");
}
```
```
